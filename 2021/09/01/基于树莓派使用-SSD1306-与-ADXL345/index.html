基于树莓派使用 SSD1306 与 ADXL345

Czz

2021-09-01 16:36:48

配件清单：

  • 树莓派4B 8G版本
  • ADXL345（ 一个加速度计可以用来检测XYZ三轴的加速度 ）
  • SSD1306 （ 一个 128 * 32 的OLED显示装置 ）

想法：

  • 通过 ADXL345 进行一些交互操作并通过 SSD1306 进行程序的切换；
  • SSD1306 显示一些已有的程序，如：显示树莓派状态以及IP等；

前言

​ 由于在家太无聊，正好一个披着红色亚克力外壳的 8G 树莓派4B告诉我，觉得他的职能不仅仅是部署一个基于Hexo静态页面的个人博客网站（ 当然实际上我仅仅使用它进行主观操作真正部署该服务器的是GitHub 具体可以参考我的有关博客搭建的 Blog ），于是就想到能不能将通过加速度计作为交互的方式，对一块 OLED 屏幕进行操作；

ADXL345

​ 这个传感器是 ADI 公司推出的基于 iMEMS 技术的 3 轴、数字输出加速度传感器。具体选择它的原因是它同时支持 SPI 和 I2C 两种通信方式，而我需要一块树莓派驱动包括一个小风扇、上面提到的SSD1306、ADXL345，担心串口不够但实际上仅仅使用了两个I2C就把问题解决了。

​ 其特点也包括：

  • 分辨率高。最高 13 位分辨率。
  • 灵敏度高。最高达 3.9mg/LSB，能测量不到 1.0°的倾斜角度变化。
  • 尺寸小。（ 方便以后将其贴到板子上 ）

具体引脚图以及功能就不再展示了，硬件方面完全不懂。

通过 Adafruit 的 ADXL345 库就可以进行简单的循环打印加速度值的程序；

SSD1306

​ 128 * 32 的OLED屏幕，同样也可以通过 I2C 的通信方式使用树莓派进行控制其显示内容；

​ 详细信息就不过多描述了；

​ 使用 Adafruit 的 SSD1306 库进行测试；

如何同时驱动它们？

​ 这里我选择的是通过两个I2C的通信方式，树莓派有两对 SDL、SCL 的串口，但需要进行开启，非图形界面使用命令 sudo raspi-config 进行配置，但是我们需要用到两条总线因此，使用 i2cdetect -l 进行查看发现只有 i2c - 0 一条，经过查资料发现在 /boot/config.txt 文件中有 dtparam=i2c_arm 和 dtparam=i2c_vc 两个参数，因此推测 vc 控制的便是 i2c - 1 的总线，取消注释后，再进行 i2cdetect -l 就可以看到了两个总线信息；
